## üì¶ Installation

### Prerequisites
- Node.js 18+ and npm/pnpm
- A Supabase account (free tier works great)

### Setup Steps

1. **Clone the repository**
```bash
git clone <your-repo-url>
cd Task_Rounak
```

2. **Install dependencies**
```bash
npm install
# or
pnpm install
```

3. **Configure Supabase**

The app uses Supabase for authentication and data storage. The Supabase credentials are already configured in `/utils/supabase/info.tsx` (autogenerated by Figma Make).

4. **Run development server**
```bash
npm run dev
```

The app will open at `http://localhost:5173`

5. **Build for production**
```bash
npm run build
```

This creates an optimized production build in the `/dist` folder.

## üåê Deployment to GitHub Pages

### Option 1: Manual Deployment

1. **Build the project**
```bash
npm run build
```

2. **Create a GitHub repository** (if you haven't already)

3. **Enable GitHub Pages**
   - Go to your repository settings
   - Navigate to "Pages"
   - Select "Deploy from a branch"
   - Choose "main" branch and "/dist" folder
   - Save

4. **Push the dist folder**
```bash
git add dist -f
git commit -m "Deploy to GitHub Pages"
git push origin main
```

Your app will be live at `https://<username>.github.io/Task_Rounak/`

### Option 2: GitHub Actions (Recommended)

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: '18'
          
      - name: Install dependencies
        run: npm ci
        
      - name: Build
        run: npm run build
        env:
          NODE_ENV: production
        
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./dist

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

## üóÑÔ∏è Database Structure

The app uses Supabase's KV store to manage all user data:

### Data Schema

```typescript
// User Habits
Key: `user:{userId}:habits`
Value: Habit[] // Array of habit objects

// Daily Progress
Key: `user:{userId}:dailyData`
Value: Record<dateString, DailyData> // Date-keyed daily data

// Current Streak
Key: `user:{userId}:currentStreak`
Value: number // Consecutive days count
```

### Habit Interface
```typescript
interface Habit {
  id: string;
  name: string;
  category: 'Health' | 'Study' | 'Mental' | 'Lifestyle';
  time?: string;
  streak: number;
  completedDates: string[];
}
```

### Daily Data Interface
```typescript
interface DailyData {
  completedHabits: string[]; // Array of habit IDs
  mood: number | null; // 1-5 scale
  reflection: string; // Daily journal entry
}
```

## üîß Configuration

### Base Path
The app is configured for GitHub Pages deployment at `/Task_Rounak/`. To change this:

1. Edit `vite.config.ts`:
```typescript
base: process.env.NODE_ENV === 'production' ? '/your-repo-name/' : '/',
```

### Theme Customization
Modify theme colors in `/src/styles/theme.css`:
- Light mode: `:root` selector
- Dark mode: `.dark` selector

### Environment Variables
No `.env` file needed! Supabase credentials are managed automatically through `/utils/supabase/info.tsx`.

## üì± Usage

### Creating an Account
1. Click "Sign Up" on the login page
2. Enter your name, email, and password
3. You'll be automatically logged in

### Adding Habits
1. Click the "+" button (desktop sidebar or mobile floating button)
2. Enter habit name, select category, and optional time
3. Click "Add Habit"

### Tracking Progress
1. Check off habits as you complete them
2. Your streak updates automatically
3. View progress in the top bar

### Viewing Insights
1. Navigate to "Insights" in the sidebar
2. See weekly, monthly, and yearly charts
3. Export data to Excel for offline analysis

### Exporting Data
1. Go to Insights page
2. Scroll to "Export to Excel" section
3. Select date range
4. Click "Export Excel File"

## üêõ Troubleshooting

### Build Errors
- Clear `node_modules` and reinstall: `rm -rf node_modules && npm install`
- Clear Vite cache: `rm -rf node_modules/.vite`

### Authentication Issues
- Check Supabase dashboard for auth errors
- Verify email confirmation is disabled (or setup email service)
- Check browser console for specific error messages

### Deployment Issues
- Ensure `base` path matches your GitHub repository name
- Check that GitHub Pages is enabled in repository settings
- Verify the build completed successfully
